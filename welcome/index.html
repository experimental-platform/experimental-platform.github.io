Experimental Platform

the easiest way to

Experimental Platform provides an easy way to deploy apps on local hardware without the hassle to setup the environment. It enables developers and makers to focus on the things that matter.

Our 5 Minute Promise

With Experimental Platform you'll be able to build your first hardware integration in under 5 minutes - using standard web technologies.

Here is our example project: Connect a CO₂ Sensor to this Smartlight.
Control the Lightbulb depending on the CO₂ levels.

If you aren't already running the experimental platform, you will need to set it up. Luckily this is very easy:

Setup experimental platform

Installing platform and creating your first app only takes a couple of minutes.
Your app will instantly be available on the internet while it's still hosted on your local machine.

  1. Connect to the CO₂ Sensor

    Connecting to the CO₂ Seonsor is very easy. You can read out the CO₂ levels via a simple HTTP request:

    GET host:3200/devices/:sensorname

    Replace the host with the address of the machine the experimental platform is running on.

  2. Connect to the Philips Hue

    Controlling the Philips Hue lightbulb couldn't be easier, just use an http request to control the lightbulb:

    PUT: hue.local/api/:token/lights/:lights_id/state

  3. See it in action

    We prepared an example app to demonstrate all the different parts acting together to create an actually useful app: Change the color of the lightbulb depending on the CO₂ levels. Check it out on github .

    git clone https://github.com/experimental-platform/example-co2-to-hue.git && cd example-co2-to-hue

  4. Deploy it

    Now deploy the app from your computer:

    git remote add platform ssh://dokku@your-box.local:8022/example-co2-to-hue git push platform master

  5. That's it! Your app is now available on the internet and within your local network.

    Locally via http://your-name.local/example-co2-to-hue/ and remotely via https://your-name.protonet.info/example-co2-to-hue/

Outlook

  1. REST API

    A standard for the internet-of-things (IoT)? It's already here: REST. So no need to reinvent the weel. EP (aims to) provide(s) an easy way to access (or wrap) all of a smart devices capability. Wether it's reading (sensing) sensors or actuating actuators.

  2. Device Discovery

    Working on IoT projects shouldn't mean spending hours trying to find your devices, getting them to connect and figure out their APIs. EP comes with a simple device discovery mechanism that is both simple to use and simple to expand.

  3. Firmware Directory

    That CO2 sensor you always wanted to build and hook up to your wireless network? Somebody already built that! By building up a directory of firmwares and an easy way to flash your devices we'll be able to build upon each others work. That's what it's all about isn't it?

  4. Services

    That cool app that you just pushed to the experimental platform now needs a simple messaging layer? What if you could simply add a service to provide that? Just like on Heroku, EP provides a simple way to include services and use them in your application.

Use your favorite technology

Use any language you like. Since the interface to the hardware is via http you can use any technology and language you prefer to write your apps for the experimental platform.

This project wouldn't be possible without the extensive use of other great open source technologies: Docker, Dokku, XYZ, Node.js, Go, abc and so many more. The code is open source under Apache 2.0. Feel free to build, fork and share.
